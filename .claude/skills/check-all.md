# check-all 技能

运行所有类型检查和代码质量检查，确保代码可以提交。

## 使用场景

在提交代码前或 PR 创建前，运行全面检查以确保代码质量。

## 技能执行指令

1. **运行类型检查**
   ```bash
   pnpm check
   ```
   这会对所有 workspace 运行 TypeScript 类型检查

2. **运行 Lint 检查**
   ```bash
   pnpm lint
   ```
   这会检查所有代码是否符合 ESLint 规范

3. **分析结果**
   - 如果所有检查通过，可以安全提交代码
   - 如果有错误或警告，需要修复后重新检查

## 输出说明

- ✅ **类型检查通过** - 所有 TypeScript 类型正确
- ✅ **Lint 检查通过** - 代码符合规范
- ❌ **发现问题** - 列出所有错误和警告，按文件分组

## 常见问题处理

如果检查失败：

1. **类型错误** - 检查 TypeScript 错误信息，修复类型问题
2. **Lint 错误** - 运行 `pnpm lint:fix` 自动修复可修复的问题
3. **构建依赖问题** - 确保已运行 `pnpm build` 构建 shared 包

## 注意事项

- 此技能不会修改代码，只做检查
- 建议在每次提交前运行
- 如果修改了 shared 包，需要先 `pnpm -C packages/shared build`
